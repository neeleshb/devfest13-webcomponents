Web Components

What are these and what do they do?

  • Today, widgets build out of HTML and JS suffer from a problem that they lack encapsulation. This means that your document stylesheet might accidentally apply to parts inside the widget; your JavaScript might accidentally modify parts inside the widget; your IDs might overlap with IDs inside the widget; and so on.

  • Web Components precisely solve this problem.

  • Web Components is a set of cutting edge standards that make it possible to build widgets, which can be reused reliably, which are encapsulated from JavaScript or stylesheet running on your pages, and which won’t break pages if the next version of the component changes internal implementation details.

Web Components

building blocks

  • Templates:  Allow users to write an HTML markup that can be customized and reused as per the needs.

  • Shadow DOMs:  Add the power of encapsulation inside an HTML document

  • Elements:  Empower the users with the ability of writing new elements that inherit from existing elements

  • HTML Imports:  Make it feasible to import an html document inside another html document

Templates

Provide inert markups to be used later

<template id="mytemplate">
    <img src="foo.png" />
    <script>alert('foo');</script>
    <div>hello!</div>
</template>
  • We are working with DOM: No side effects of directly using HTML.
  • Templates are inert - parsed but not executed.
  • Templates are Document Fragements (i.e: document object that don't have a parent, and may contain ill-formed markups).
  • Templates are hidden from the rest of the document. We cannot traverse directly into their DOM.

Creating a Shadow Dom

Theory

  • Start with the node that you wish to encapsulate.
  • create a shadow root by calling createShadowRoot method on that node.
  • Replace the innerHTML of the shadow root by whatever content you want. This innerHTML gets encapsulated and is not accessible by the outside DOM
  • The original node that is shadow'ed gets rendered based on what innerHTML you added, not based on its original content.
  • There is no way for any external JS to access this newly added innerHTML. Any attempt to access the contents of the original node will return the "original content" present in the source, not the content defined by the shadow.

Creating a Shadow Dom

Code

<div id="host">
    <h2>My Header</h2>
    <p>My paragraph</p>
</div>
<script>
    var hostElem = document.querySelector("#host");
    var shadow = hostElem.webkitCreateShadowRoot();
    shadow.innerHTML = 
      "<h2>My new header</h2><p>my new paragraph</p>";	
</script>
<button onclick="alert(document.querySelector('#host').innerHTML);">
    click me
</button>

Custom Elements

Inheriting from existing elements

Custom Elements

Aren't they alredy supported?

<body>
    <random onclick="alert(1);"
        style="border:1px solid red;width:200px;height:200px;display: block;">
    </random>
</body>
  • All unknown elements inherit from HTMLUnknownElement, which is a subclass of HTMLElement.
  • No useful functionality is associated with an HTMLUnknownElement, so an HTMLUnknownElement cannot behave like a button or a textbox or a video tag etc.
  • That is the reason why we need Custom Elements. They can inherit from known HTML Elements like HTMLButtonElement etc, and thus get their properties automatically.

Custom Elements

Defining and registering

  • Decide a name for your custom element. For example - "x-foo", "mega-button", "email-text" etc. The name must contain a hyphen.

  • Decide which HTML element you want your custom element to inherit from. For example, if your custom element "mega-button" is a specialized button, it should inherit from HTMLButtonElement. Similarly a textarea might inherit from HTMLTextAreaElement and a text input might inherit from HTMLInputElement.

  • Register the new element by using document.register
    var MegaButton = 
    document.register('mega-button', {
        prototype: Object.create(HTMLButtonElement.prototype)});

Custom Elements

Three ways of instantiating

  • Use the HTML Markup

    <button is="mega-button" />

  • Use DOM function document.createElement

    var megaButton = 
    document.createElement('button', 'mega-button');
document.body.appendChild(megaButton);

  • Use the new operator

    var megaButton = new MegaButton();
document.body.appendChild(megaButton);

Custom Elements

Adding properties and methods

<button is="mega-button" onclick="this.foo();">Mega Button</button>
<button>Normal Button</button>

<script>
    var megaButtonProto = Object.create(HTMLButtonElement.prototype);
    megaButtonProto.foo = function() {
        alert("Mega button clicked");
    };

    document.register('mega-button', {prototype: megaButtonProto});
</script>

Custom Elements

Lifecycle Callbacks

  • createdCallback: When a new instance of the element is created.

  • enteredViewCallback: When an instance of the element gets inserted in the document

  • leftViewCallback: When instance of the element gets removed from the document

  • attributeChangedCallback(attrName, oldVal, newVal): When an attribute gets added, removed, or updated in the instance of the element
